## 一、项目概述

### 1.1 解决的教学问题

小学数学中**抽象概念难理解**——几何图形的性质、分数的意义、平移旋转的变换思想、线段图的数量关系。传统教学依赖静态图示和动画播放，学生只能"看"不能"动"，缺乏亲手操作、反复探索的机会。

### 1.2 解决方案

开发一个**基于DeepSeek大模型的Web智能导学系统**，包含四大动态演示模块，学生可在Canvas画布上自由拖拽操作，LLM实时充当智能导学角色：根据操作引导思考、诊断错误概念、自适应出题、多角度讲解、激发探究。

### 1.3 核心创新

- **操作即对话**：学生拖拽图形 → 系统感知变化 → LLM生成针对性追问，形成"操作-思考-反馈"闭环
- **用AI开发AI工具**：整个系统借助DeepSeek辅助编码完成，开发过程本身是"创AI"的最佳证据
- **纯前端零部署**：HTML+JS单文件结构，托管后一个链接即可使用，对软硬件要求极低

## 三、四大模块详细设计

### 模块一：几何图形探索

**教学目标**：理解三角形/四边形的分类（等腰、直角、平行四边形等）、内角和、面积推导

**画布元素**：
- 可拖拽顶点的三角形（至少1个，支持添加/删除顶点变成四边形）
- 实时显示：角度数值（弧标记）、边长数值
- 辅助线开关：高、中线、对角线

**交互操作**：
1. 拖拽顶点 → 角度/边长同步变化
2. 双击画布 → 新增顶点（三角形变四边形）
3. 右键顶点 → 删除

**LLM介入时机**：
- 学生拖出一个等腰三角形 → "两个底角显示相同的度数，你发现它们总是一样大吗？试试拖拽顶点验证一下"
- 学生拖出让三角形内角和接近180° → "三个角加起来刚好多少度？试试任意拖拽，这个和会变吗？"
- 学生操作后无明显发现 → "试试把B点水平向右拖，观察AC边上的高是怎么变化的？"

**关键状态参数**（传给LLM）：
```json
{
  "shape": "triangle",
  "vertices": [[x1,y1],[x2,y2],[x3,y3]],
  "angles": [45, 60, 75],
  "sides": [3.2, 4.1, 5.0],
  "angleSum": 180,
  "isIsosceles": true,
  "operation": "dragged_vertex_B"
}
```

### 模块二：分数认识

**教学目标**：理解分数的份数含义、等值分数、分数大小比较

**画布元素**：
- 圆形（默认）或长方形，带N等分分割线
- 滑块控制分母（2-12）
- 点击填色选中份数（分子）
- 双图形模式：左右并列两个图形用于比较

**交互操作**：
1. 拖动滑块切分 → 图形实时显示N等分
2. 点击份数填色 → 分子逐份变化
3. 双图模式下分别操作 → 直观比较1/3 vs 1/4

**LLM介入时机**：
- 学生将圆切成4份填了2份 → "你表示的是2/4。试试切8份填4份，看看和2/4的大小一样吗？"
- 学生比较1/3和1/4，认为1/4更大 → 识别"整数负迁移"错误 → "你想的是4>3对不对？但分母是分成了几份哦。你看，分4份的每一份比分3份的……"
- 双图都显示1/2 → "虽然是1/2，但一个5等分填了2.5份，一个6等分填了3份——你能用分数分别表示它们吗？"

**关键状态参数**：
```json
{
  "mode": "single" | "dual",
  "left": { "shape": "circle", "denominator": 4, "numerator": 2, "value": 0.5 },
  "right": { "shape": "circle", "denominator": 8, "numerator": 4, "value": 0.5 },
  "operation": "filled_slice",
  "studentComparison": null
}
```

### 模块三：平移与旋转

**教学目标**：理解平移（方向+距离）、旋转（中心+角度+方向）、轴对称

**画布元素**：
- 预设图形（三角形/长方形/L形/点阵图）
- 平移模式：拖拽图形整体移动，显示方向箭头和位移距离
- 旋转模式：拖拽旋转手柄，显示旋转角度弧线
- 网格背景（可选开关）

**交互操作**：
1. 平移：拖拽图形到任意位置，虚线显示原始位置
2. 旋转：拖拽旋转手柄，绕中心点旋转
3. 切换模式：平移/旋转/轴对称（显示镜像）

**LLM介入时机**：
- 学生旋转90°后 → "旋转后图形变了吗？形状、大小、方向，哪些变了哪些没变？"
- 学生做平移后 → "向上平移3格再向右平移5格，如果反过来先右再上，结果一样吗？试试看"
- 学生操作对称 → "对称轴在这里，你能预测B点的对称点在哪吗？先猜再操作"

**关键状态参数**：
```json
{
  "mode": "translate" | "rotate" | "reflect",
  "originalPosition": { "x": 100, "y": 200 },
  "currentPosition": { "x": 250, "y": 200 },
  "translation": { "dx": 150, "dy": 0 },
  "rotationAngle": 0,
  "gridSnap": true
}
```

### 模块四：线段图

**教学目标**：理解数量关系（和差倍分），将文字题转化为线段图

**画布元素**：
- 可拖拽的线段（水平条），两端和中间可拖动调整长度
- 多段并列（甲、乙两段），支持标注数值
- 自由添加新段

**交互操作**：
1. 拖拽线段端点 → 调整长度
2. 拖拽整段 → 移动位置
3. 旁边实时显示"甲是乙的X倍"等关系

**LLM介入时机**：
- 学生建好线段图 → "已知甲是乙的3倍，甲+乙=36，你能用线段图表示出来吗？"
- 学生比例不对 → "你说甲是乙的3倍，但甲这条是乙的2倍长。试试把甲拉长到恰好3个乙那么长"
- 自动出题 → "看看你画的图：甲乙共5份，如果总长是30cm，每份是多少？甲占几份？甲长多少？"

**关键状态参数**：
```json
{
  "segments": [
    { "id": "A", "length": 300, "label": "甲" },
    { "id": "B", "length": 100, "label": "乙" }
  ],
  "ratio": "3:1",
  "totalLength": 400,
  "operation": "adjusted_A_length"
}
```

## 四、LLM对话系统设计

### 4.1 触发策略

| 触发方式 | 说明 |
|---------|------|
| **被动触发** | 学生点击"问老师"按钮，或直接在对话框输入问题 |
| **主动触发** | 系统监测到有意义的操作（拖拽完成/形成特殊图形/停留30秒无操作），LLM自动发起引导 |
| **错误触发** | 检测到典型错误模式（如分数比较中整数负迁移），LLM主动介入纠正 |

### 4.2 System Prompt 核心设计

```
你是小学数学智能导学助手"小Q"，你的角色是苏格拉底式引导者：
- 不直接给答案，通过提问引导学生自己发现
- 用具体的操作引导代替抽象解释（"试试把顶点往上拉"而非"改变y坐标"）
- 识别学生的典型错误概念，给予针对性引导
- 用小学生能理解的比喻和语言（"份数"而非"分母"，"转圈"而非"旋转角度"）
- 及时肯定学生的发现，激发进一步探究
- 你当前面对的是小学[X]年级学生

当前模块：[几何图形/分数/平移旋转/线段图]
当前操作状态：[JSON状态数据]
```

### 4.3 对话安全

- 对话内容限定在小学数学和当前模块范围内（防止越狱滥用）
- API Key 通过后端代理，不暴露在前端
- 对话历史不超过10轮（控制Token消耗）

### 4.4 成本估算

- DeepSeek API：约 ¥1/百万Token
- 单次对话（含状态上下文）：约 500-1000 Token
- 预估每学生每节课消耗成本：< ¥0.01

## 六、开发实施计划

### 阶段一：基础框架搭建（预计2-3天）

1. 创建项目文件结构
2. 实现HTML布局框架（模块切换 + Canvas画布 + 对话面板）

### 阶段二：四个模块并行开发（预计3-5天）

1. 模块一：几何图形（顶点拖拽、角度/边长实时计算）

1. 为每个模块设计专用System Prompt

1. 全流程测试（操作→LLM响应→继续操作）
2. 性能优化（Canvas重绘效率、API响应等待动画）
3. 移动端适配
